# CEO-чеклист: local готов / можно на прод

Короткий статус для решения «ещё готовим» vs «можно смотреть живой сайт».  
Без технического жаргона. Копируется в любой проект.

Агент SEO+GEO-аудита **заполняет этот блок в отчёте** (отметить ✅ / ❌ / —), особенно после прогона **local**.

---

## Статус проекта

| Статус | Когда ставить | Что делать дальше |
|--------|---------------|-------------------|
| **Ещё готовим (local)** | Не закрыты обязательные пункты этапа 1 | Чинить локально, прод не трогать |
| **Можно выкладывать** | Этап 1 закрыт | Выкладка → сразу этап 2 |
| **Живой сайт проверен** | Этап 2 закрыт | Только точечные улучшения |

---

## Этап 1 — Local готов (закрыть до выкладки)

Проверка на **локальной / preview**-версии. Боевой сайт не нужен.

### Обязательно (без этого не выкладываем)

- [ ] Главная открывается и выглядит как задумано
- [ ] Главная коммерческая страница (заявка / услуга / покупка) открывается и ведёт к действию
- [ ] Заголовки страниц и короткие описания в поиске заполнены на ключевых страницах (не «Untitled», не пусто)
- [ ] Нет запрета «не показывать в поиске» на страницах, которые **должны** быть в поиске
- [ ] Карта сайта и правила для поисковиков настроены под будущий боевой адрес (если уже известен)
- [ ] Ссылки внутри сайта ведут куда нужно (нет битых меню и «пустых» разделов на ключевых путях)
- [ ] Тексты ключевых страниц понятны человеку: ясно кто вы, что предлагаете, что делать дальше
- [ ] На ключевых страницах есть короткие понятные ответы (что это / для кого / в чём польза) — чтобы и человек, и нейросеть могли процитировать суть
- [ ] На мобильном ключевые страницы читаются и кнопки нажимаются

### Желательно до выкладки

- [ ] Карточки для соцсетей (превью ссылки) выглядят нормально
- [ ] Контакты / доверие (адрес, телефон, «о нас») на месте, если это важно для ниши
- [ ] Нет явных дублей одной и той же страницы под разными адресами в локальной версии
- [ ] В правилах для роботов нет случайного полного запрета для ботов ИИ (если видимость в ответах нейросетей важна бизнесу)
- [ ] По главным запросам посмотрели выдачу: есть ли блок ответа нейросети и **какие сайты** там фигурируют (рынок, не «наш» индекс)
- [ ] Кратко сравнили 2–3 сильных конкурента: у кого ответы и факты лучше «цитируются» — и закрыли пробелы в **наших** текстах до запуска
- [ ] Тексты ключевых страниц прошли чеклист готовности к ответам ИИ (см. `geo-content-checklist.md`)

**Решение этапа 1**

- Все обязательные пункты ✅ → статус **Можно выкладывать**
- Есть ❌ в обязательных → статус **Ещё готовим (local)** — сначала закрыть их

## Этап 2 — Можно на прод / живой сайт проверен

Только **после выкладки**, на **вашем** боевом адресе. Сюда не кладём «посмотреть конкурентов» — это уже этап 1.

### Обязательно (первый проход после выкладки)

- [ ] Живой адрес открывается по https, без лишних редиректов «по кругу»
- [ ] Главная и главная коммерческая страница на бою совпадают с тем, что согласовали локально
- [ ] Правила для поисковиков и карта сайта доступны на боевом адресе
- [ ] Страницы, которые должны быть в поиске, **не** закрыты от индексации на бою
- [ ] Адреса страниц (канонические) указывают на боевой сайт, а не на localhost
- [ ] Превью в мессенджерах / соцсетях с боевого адреса выглядят нормально

### В течение первых дней / недель (по возможности)

- [ ] Подключён доступ к отчётам поиска (кто видит **ваш** сайт в Google) — если ещё не было
- [ ] Нет массовых ошибок «страница не найдена» на ключевых разделах
- [ ] Скорость на реальных пользователях приемлема (если есть данные) — иначе поставить задачу измерить
- [ ] Репутация и ссылки на **ваш** домен не выглядят токсично (базовая проверка доверия)
- [ ] По возможности: зонд — упоминают ли нейросети **ваш** адрес по целевым вопросам (это не гарантируется сразу)

**Решение этапа 2**

- Обязательные пункты ✅ → статус **Живой сайт проверен**
- Есть ❌ → статус остаётся «выложено, но не принято» — чинить на бою или откатить спорное

---

## Как читать в одном взгляде

```text
Этап 1 (local):  [ ] не готов   [ ] можно выкладывать
Этап 2 (prod):   [ ] не начинали   [ ] в работе   [ ] проверен
```

Правило Monster Automation: **сначала этап 1, потом выкладка, потом этап 2.**  
Не смешивать: закрытый local ≠ проверенный прод.

## Связь с командой

| Ситуация | Команда |
|----------|---------|
| Готовим сайт | `/MA-seo-geo-audit local` (или с `http://localhost:…`) |
| Только что выложили | `/MA-seo-geo-audit <боевой-url> post-deploy` или `quick prod` |
| Нужен только этот чеклист в отчёте | агент заполняет статусы по факту аудита |

После закрытия P0 на local: выкладка → `/MA-seo-geo-audit <url> quick prod`.
