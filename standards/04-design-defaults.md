# 04 · Design defaults

**Версия стандартов:** см. `VERSION`

## Два контекста

### A. Админка / дашборд / портал
- Плотность, сканируемость, таблицы и формы
- Типографика: чистая системная/Geist-подобная; без «маркетингового» display в таблицах
- Карточки — осторожно; списки и панели чаще лучше
- Следовать `02-product-preferences.md`

### B. Маркетинг / лендинг / промо
- Brand-first на первом экране
- Один сильный визуальный якорь (реальный продукт/место/атмосфера)
- Без карточного шума в hero
- Anti-slop правила (см. user frontend design rule + impeccable skill)

## Общее
- WCAG AA контраст как цель
- Фокус-кольца видимые
- Не полагаться только на цвет для статуса
- Motion: 2–3 намеренных движения, не шум
- shadcn: базовые примитивы; кастомизация через токены проекта

## Ритуал экрана
Для любого нового/переделанного экрана: `/MA-design-screen`  
Он обязан читать этот файл + `02-product-preferences.md`.
