# 03 · Engineering defaults

**Версия стандартов:** см. `VERSION`

## Стек по умолчанию (если проект не говорит иное)
- Web apps: **Next.js App Router** + TypeScript + Tailwind
- UI kit: **shadcn/ui** + Lucide
- Backend data: **Supabase** (Auth / DB / RLS) — если уже в проекте или выбран
- Deploy: **Vercel** — если уже в проекте или выбран
- Не менять стек ради «моды» без бизнес-причины

## Качество кода
- TypeScript strict where project already uses it
- Серверные проверки прав и валидации — обязательно
- Секреты только в env / secret store, не в git
- Не коммитить без явной просьбы пользователя
- Не пушить / force-push без явной просьбы

## Производительность (базовый бар)
- Не грузить лишнее на первый экран
- Изображения: размеры и lazy где уместно
- Клиентские бандлы: не тащить тяжёлые lib без нужды

## База и данные
- Миграции версионируются
- RLS / политики доступа — часть фичи, не «потом»
- Индексы под реальные фильтры списков/канбана

## Тесты
- Критичные потоки (auth, деньги, права): минимум smoke / e2e по возможности
- Не требовать 100% покрытия ради галочки
