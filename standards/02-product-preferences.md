# 02 · Product preferences (ваши «хотелки»)

**Версия стандартов:** см. `VERSION`

Это ваш единый файл предпочтений вместо разрозненных заметок.

---

## Дизайн-компас (главная хотелка)

**Все дизайны наследуют один язык:**

- **Токены и принципы** — как у **Vercel** и **Linear** (точный tool-UI, спокойная нейтральная база, Geist, плотность, минимум декора)
- **Удобство** — как у **Apple HIG** (ясность, последовательность, обратная связь, доступность)

Подробности и чеклист: `04-design-defaults.md`.  
Новый экран без этого компаса = отклонение от стандарта (нужно явное «делаем иначе»).

## Дашборды и админки
- Плотность: высокая, сканируемость важнее «воздушного маркетинга»
- Первый экран: одна главная задача пользователя, без hero-баннеров внутри портала
- Карточки: только если держат действие или сущность; не карточный шум
- Таблицы: основной паттерн списков (сортировка/фильтры), акцент на whitespace, без лишних внутренних бордеров
- Статусы: бейджи спокойные; не кодировать смысл только цветом
- Empty / loading / error: обязательны и понятны нетехническому пользователю

## Канбан
- Колонки = явные статусы процесса (названия бизнес-языком)
- Карточка: заголовок, статус/приоритет, владелец, срок (если есть) — без перегруза
- Drag-and-drop предсказуемый; отмена/ошибка не теряет данные
- Фильтры и поиск сохраняются в URL (можно вернуться по ссылке)
- Мобилка: сначала список/сжатый вид, полный канбан — desktop-first ok, но не ломать

## Авторизация и доступ
- Явный вход / выход; сессия понятна пользователю
- Роли и права: нельзя открыть чужие данные подбором id (обязательная проверка на сервере)
- Ошибки доступа: понятное сообщение, без утечки лишнего
- Восстановление доступа: отдельный понятный поток (если продукт это требует)
- 2FA / SSO: подключать когда бизнес скажет; заложить возможность без оверинжиниринга

## Формы
- Валидация с понятными сообщениями и следующим шагом
- Не блокировать вставку в поля
- Кнопка submit: ясный глагол («Сохранить клиента», не «Продолжить»)
- Несохранённые изменения — предупреждение перед уходом

## Навигация
- Структура под задачи ролей (оператор / админ / клиент), не под внутреннюю архитектуру
- Глубокие ссылки на фильтры, вкладки, открытые панели
- Деструктивные действия — подтверждение или окно отмены

## Тон продукта
- Второй человек, активный залог
- Без emoji в UI как иконок по умолчанию (Lucide/SVG)
- Числа и даты — локаль пользователя

## Бренд и тема
- Сначала дизайн-компас (Vercel×Linear + Apple HIG), **поверх** — бренд проекта (логотип, один accent)
- Админки: нейтральная точная тема, сильный контраст текста
- Маркетинг/лендинги: brand-first, один сильный первый экран, но usability всё ещё HIG
- Anti-slop: без дефолтных фиолетовых градиентов, стеклянного шума, карточных сеток ради красоты
