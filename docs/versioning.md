# Версионирование

## standards/VERSION

Semver для содержимого `standards/`:

- **MAJOR** — ломающие требования (старые проекты надо пересмотреть)
- **MINOR** — новые предпочтения без ломки
- **PATCH** — формулировки и уточнения

После bump: `git tag vX.Y.Z` (опционально) и push. Weekly pull подхватит `main`.

## Политика для проектов

| Режим | Когда | Как |
|-------|--------|-----|
| **track main** (дефолт) | обычные продукты | `ma-hub-pull` / weekly; revise читает текущий `~/ma-hub` |
| **pin tag** | критичный прод, не хотим сюрпризов | в `docs/MA-STANDARDS.md`: `Pin: v1.0.0`; pull с `MA_HUB_REF=v1.0.0` |

Проекты **не хранят копию** стандартов целиком — только ссылку и режим. Иначе копии устаревают.

## Команды

Команды в `commands/` версионируются вместе с хабом. `install-commands.sh` всегда копирует актуальные файлы в `~/.cursor/commands/`.
