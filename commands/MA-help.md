# MA-help — команды Monster Automation

Покажи этот список коротко и предложи, какую команду запустить. Все команды начинаются с **MA-** — в чате набери `/MA` и выбери нужную.

## Hub (источник правды)

Сначала найди корень **ma-hub**:
1. `~/.config/ma-hub/config` → `MA_HUB_ROOT`
2. иначе `~/ma-hub`, если есть `standards/`
3. иначе скажи запустить bootstrap: `git clone … && ./bootstrap/bootstrap.sh`

Стандарты: `$MA_HUB_ROOT/standards/` (VERSION в том файле).

## Ритуалы

| Команда | Когда вызывать |
|---------|----------------|
| `/MA-idea-to-plan` | Сырая идея / фича → уточнения → план → работа |
| `/MA-design-screen` | Админка, дашборд, лендинг, красивый фронт |
| `/MA-revise-project` | Сверить проект со стандартами ma-hub и подтянуть |

## Релиз

| Команда | Когда вызывать |
|---------|----------------|
| `/MA-deploy` | Полный цикл перед продом (проверки → push → CI → PR) |

## Справка

| Команда | Когда вызывать |
|---------|----------------|
| `/MA-help` | Этот список |

## Как пользоваться
1. Открой чат в нужном проекте
2. Напиши `/MA` и выбери команду
3. После команды опиши задачу одним-двумя предложениями
4. На новом ноутбуке: bootstrap из публичного ma-hub (см. README хаба)

**Заметка:** security / UX / guidelines — через навыки бандла или обычным текстом в чате.

Если пользователь не указал задачу — спроси одну фразу: что делаем. Напомни, что стандарты живут в ma-hub и обновляются через `ma-hub-pull` / weekly.
